The code list LC_OnTopType contains a list of types of on top relationships for the onTop attribute of the LC_InterStrataRelationship association class. The onTop relationship make use of a code list in order to make it possible to define as many values as one wants and give them explicit meanings. This is the most flexible approach to handling the onTop relation. This list contains nine types<i>.</i> This list may be extended through registration.  The attribute <i>onTopType</i> for the association class LC_InterStrataRelationship is optional and is by default the first value of this enumeration "<i>baseline</i>".The on top relationship types are:<br/><br/><b>baseline</b> - Elements of a stratum are "on top" of the baseline, where the baseline is a reference surface which corresponds to the surface of the earth (or water) covered by the land cover features that are instantiations of the LC_LandCover metaobject. Baseline is the default value of the relationship <i>onTop</i>.  <br/><br/><b>previousE1</b> - Elements of a stratum are "on top" of the first element (or set of elements regulated by an exclusive, temporal or conditional relationship) of the previous stratum. <br/><br/><b>previousE2</b> - Elements of a stratum are "on top" of the second element (or set of elements regulated by an exclusive, temporal or conditional relationship) of the previous stratum. <br/><br/><b>previousE3</b> - Elements of a stratum are "on top" of the third element (or set of elements regulated by an exclusive, temporal or conditional relationship) of the previous stratum. <br/><b><br/></b><b>previousE1-2</b> - Elements of a stratum are "on top" of the first to second element of the previous stratum (where either the first and /or second element may be a set of elements regulated by an exclusive, temporal or conditional relationship). <br/><br/><b>previousAll</b> - Elements of a stratum are "on top" of the all the elements of the previous stratum. <br/><br/><b>ptratum1E1</b> - Elements of a stratum are "on top" of the first element (or set of elements regulated by an exclusive, temporal or conditional relationship) of the first stratum defined in the set of stratum which are components of the LC_LandCover metaobject.  <br/><br/><b>ptratum1All</b> - Elements of a stratum are "on top" of all of the elements of the first stratum defined in the set of stratum which are components of the LC_LandCover metaobject.  <br/><br/><b>ptratum2E1</b> - Elements of a stratum are "on top" of the first element (or set of elements regulated by an exclusive, temporal or conditional relationship) of the second stratum defined in the set of stratum which are components of the LC_LandCover metaobject.  <br/><br/><b>ptratum2All</b> - Elements of a stratum are "on top" of all of the elements of the first stratum defined in the set of stratum which are components of the LC_LandCover metaobject.  <br/><br/>[Note: Other relations such as PreviousE4, Previous E2-5, Stratum3E4 etc. which are logical extensions of the predefined relations can be defined through registration by the user, but are not predefined in the standard for simplicity and because such complex relations would be used infrequently.]<br/><br/>
